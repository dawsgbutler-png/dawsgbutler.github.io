AI Trading Decision System

A self-directed paper-trading platform built to test AI-assisted decision logic, risk controls, scanning, portfolio rotation, backtesting, and live monitoring.

CONTEXTIndependent Project
TIMELINE2026–present
COLLABORATIONSelf-directed with AI-assisted development
THE CHALLENGE

Explore how far I could push AI-assisted development by creating a system that evaluates a large stock universe, converts market data into structured decisions, manages risk, executes simulated orders, and explains what it is doing.

MY ROLE

I defined the system requirements, tested the logic, reviewed failures, organized modules, and iterated on the scanner, risk manager, rotation engine, execution workflow, logging, dashboard, and replay tests. The project was built in my free time to learn coding, APIs, automation, and quantitative decision systems.

PythonAlpaca APIMarket data APIsAutomationBacktestingRisk controlsDashboard design
ENGINEERING PROCESS

How the work moved from problem to functioning result.

01

Design the architecture

The system separates market data, feature generation, regime detection, scoring, risk management, portfolio rotation, execution, and logging so each part can be tested and improved.

02

Build a repeatable loop

Every cycle checks system health, reads current data, evaluates the market regime, scores candidates, reviews positions, applies risk rules, and records the outcome.

03

Test before trusting

Paper trading and historical replay testing expose weaknesses without risking real capital.

04

Improve transparency

Mission Control summarizes system status, decisions, errors, blocked trades, position changes, and performance instead of hiding the process behind a single output.

OUTCOME

The project dramatically expanded my comfort with Python, APIs, debugging, data structures, automation, and system reliability. It also taught me that a sophisticated dashboard is less important than disciplined testing and clear risk controls.

WHAT I LEARNED

AI can accelerate development, but it does not replace engineering judgment. I still have to define requirements, detect incorrect assumptions, compare results, manage edge cases, and decide whether the system is behaving logically.

Project note: This is a paper-trading and software-learning project. All displayed trades and performance are simulated or representative and are not financial advice.
